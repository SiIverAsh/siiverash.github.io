# 📝 Silverash's Blog Commit 规范指南

为了保持 GitHub 仓库历史的整洁与专业，建议采用以下提交格式：

## 📌 固定格式
`<类型>: [分类] 简短描述`

---

## 📋 常用类型定义

| 类型 | 说明 | 示例 |
| :--- | :--- | :--- |
| **post** | 发布新博客文章 | `post: [Study] 新增SSH远程连接配置指南` |
| **edit** | 修改/纠正博文内容 | `edit: [Anime] 修正《芙莉莲》漫评的错别字` |
| **snap** | 发布摄影/随拍内容 | `snap: [Snap] 记录一张校园里的猫` |
| **ui** | 美化界面、调整CSS/布局 | `ui: 调整搜索框下拉层级` |
| **feat** | 增加网站新功能 | `feat: 增加全站搜索功能` |
| **fix** | 修复Bug/显示问题 | `fix: 修复首页时钟显示不出的问题` |
| **bot** | AI脚本/自动化逻辑改动 | `bot: 优化DeepSeek打标签的提示词` |
| **chore** | 配置更新、依赖安装等 | `chore: 更新_config.yml里的社交链接` |

---

## 💡 使用小贴士
*   **简洁明了**：描述尽量控制在 50 个字符以内。
*   **英文类型**：类型（Type）建议统一使用英文小写，显得更专业。
*   **一推多篇**：如果一次推送了多篇文章，可以用 `post: [Multi] 发布三篇关于AI的笔记`。
